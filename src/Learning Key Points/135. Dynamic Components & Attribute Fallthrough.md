## Event Handling in Vue Componenten

### (HBO-niveau uitleg)

---

## 1. Inleiding

Binnen Vue.js worden componenten gebruikt om herbruikbare UI-elementen te definiëren. Een belangrijk aspect van componentgebruik is **event handling**, oftewel het reageren op gebruikersinteracties zoals klikken.

In deze context ontstaat vaak de vraag:

> Hoe kan een `@click` op een custom component automatisch werken op een onderliggend HTML-element?

Om dit te begrijpen, is het noodzakelijk om inzicht te hebben in het concept **event forwarding** en de rol van het **root element** binnen een component.

---

## 2. Basisvoorbeeld

### Gebruik van een component

```vue
<base-button @click="selectTab('stored-resources')">
  Stored Resources
</base-button>
```

### Definitie van de component

```vue
<template>
  <button :type="type" :class="mode">
    <slot></slot>
  </button>
</template>
```

---

## 3. Het Root Element Principe

Elke Vue component heeft (in de meeste gevallen) één **root element**.

In het voorbeeld:

```vue
<button>...</button>
```

is de `<button>` het root element van de component.

### Definitie

> Het **root element** is het bovenste DOM-element dat door een component wordt gerenderd.

---

## 4. Automatische Event Forwarding

Wanneer een event listener wordt toegevoegd aan een component:

```vue
<base-button @click="...">
```

dan voert Vue intern het volgende gedrag uit:

> Vue koppelt deze event listener automatisch aan het root element van de component.

### Intern gedrag (conceptueel)

```vue
<button @click="...">
```

Dit mechanisme wordt aangeduid als:

### ➤ **Event Forwarding (of Attribute Inheritance)**

---

## 5. Waarom werkt dit?

Dit werkt omdat:

1. `click` een **native DOM event** is
2. Vue herkent dat het component slechts één root element heeft
3. Vue automatisch attributen en events doorgeeft aan dat root element

---

## 6. Native Events vs Component Events

### Native DOM events

Voorbeelden:

- `click`
- `input`
- `focus`

Deze kunnen direct op HTML-elementen worden toegepast:

```vue
<button @click="...">
<div @click="...">
```

### Component events

Wanneer je zelf events definieert:

```vue
<button @click="$emit('custom-event')">
```

Dan moet je expliciet gebruik maken van:

> `$emit()`

---

## 7. Beperkingen van Event Forwarding

### 7.1 Meerdere root elementen

Wanneer een component meerdere root elementen bevat:

```vue
<template>
  <button>Click</button>
  <span>Extra</span>
</template>
```

➡️ Vue kan niet bepalen waar de event listener geplaatst moet worden
➡️ Automatische forwarding werkt hier **niet**

---

### 7.2 Wrapper elementen

```vue
<template>
  <div>
    <button>Click</button>
  </div>
</template>
```

➡️ De `@click` wordt gekoppeld aan `<div>`
➡️ Niet direct aan `<button>`

Door **event bubbling** zal een klik op de button meestal alsnog werken, maar:

- Dit is indirect gedrag
- Kan ongewenste bijwerkingen hebben

---

## 8. Event Bubbling (belangrijk concept)

Wanneer een gebruiker klikt op een element:

```html
<button></button>
```

dan “bubbelt” het event omhoog in de DOM:

```
button → div → body → document
```

Dit verklaart waarom een click soms nog werkt, zelfs als deze niet direct op het juiste element zit.

---

## 9. Best Practice

### Wanneer automatische forwarding gebruiken?

✔ Bij eenvoudige wrapper componenten
✔ Wanneer het root element gelijk is aan het interactieve element

---

### Wanneer expliciet `$emit` gebruiken?

✔ Bij complexere componentstructuren
✔ Wanneer je controle wilt over eventgedrag

```vue
<button @click="$emit('click')">
```

---

## 10. Conceptueel Model

Een Vue component kan worden gezien als:

> Een abstractielaag boven een HTML-element, waarbij het root element het daadwerkelijke gedrag bepaalt.

Dus:

```vue
<base-button>
```

gedraagt zich functioneel als:

```vue
<button>
```

zolang:

- het component één root element heeft
- en er geen expliciete overrides zijn

---

## 11. Samenvatting

- Vue koppelt native events automatisch aan het root element van een component
- Dit mechanisme heet **event forwarding**
- Het werkt alleen betrouwbaar bij één root element
- Complexere componenten vereisen expliciet gebruik van `$emit`
- Event bubbling kan gedrag beïnvloeden, maar moet niet als primaire oplossing worden gezien
